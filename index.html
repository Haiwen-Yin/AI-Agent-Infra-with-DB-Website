Loop Engineering in This Project本项目中的循环工程
Loop definitions are stored as entities (entity_type='LOOP_DEFINITION') with metadata in loop_meta. Each run tracks iterations with plan→action→observe→evaluate→adjust. Stop conditions include max_iterations, max_tokens, and max_duration. Lifecycle hooks (PRE_RUN, ON_START, POST_ITERATION, ON_STOP, ON_FAIL, ON_TIMEOUT) enable extensible automation. v3.7.2 adds Collaborative Integration: Spec-Driven Loops derive from acceptance criteria, Task-Loop Binding auto-completes steps on loop success, Collaborative Loops create parent/child loops for group coordination with AGGREGATE evaluation, and Skill-Triggered Loops auto-start validation on skill acquire.循环定义存储为实体（entity_type='LOOP_DEFINITION'），元数据存于 loop_meta。每次运行追踪迭代：计划→行动→观察→评估→调整。停止条件包括最大迭代次数、最大Token数和最大时长。生命周期钩子支持可扩展自动化。v3.7.2 新增协同集成：规格驱动循环从验收标准派生、Task-Loop绑定循环成功时步骤自动完成、协作循环创建父子循环配合AGGREGATE评估、Skill触发循环在技能获取时自动启动验证。

Step-by-Step Cross-Modal Query跨模分步查询
Query each signal independently — vector, fulltext, relational, tag, graph — then fuse results in Python. Gives AI Agents deeper understanding of query feedback, enabling better refinement for subsequent queries.独立查询每个信号——向量、全文、关系、标签、图——然后在Python中融合结果。帮助AI Agent更深入理解查询反馈，为后续查询提供更好的支持。

The PostgreSQL edition achieves full feature parity using equivalent native capabilities: Row Security Policies (RLS) replace Oracle Data Grants for per-agent row-level isolation; PL/Python3u replaces UTL_HTTP for outbound HTTP calls; pg_cron replaces DBMS_SCHEDULER for scheduled jobs; pgvector replaces Oracle VECTOR for similarity search; tsvector/tsquery replaces Oracle Text for full-text search; Apache AGE replaces Oracle Property Graph for SQL/PGQ graph traversal.PostgreSQL 版本使用等效原生能力实现完全功能对等：Row Security Policies (RLS) 替代 Oracle Data Grants 实现按Agent行级隔离；PL/Python3u 替代 UTL_HTTP 实现出站HTTP调用；pg_cron 替代 DBMS_SCHEDULER 实现定时作业；pgvector 替代 Oracle VECTOR 实现相似度搜索；tsvector/tsquery 替代 Oracle Text 实现全文搜索；Apache AGE 替代 Oracle Property Graph 实现 SQL/PGQ 图遍历。
